# System Architecture

## User Roles

- Student / Parent
- Tutor
- Admin

## Architecture Style

Client-Server architecture:

Frontend → Handles user interaction  
Backend → Handles business logic  
Database → Stores user data, bookings, reviews  

## Booking Flow

1. User searches tutor
2. Selects available slot
3. Confirms booking
4. Tutor receives notification
5. Session is scheduled
